CSS DOM 技术

发表于 2017-07-05 | 分类于 前端

style 属性

在外部样式表里声明的样式不会进入 style 对象，在文档的 <head> 部分里声明的样式也是如此。

DOM style 属性提取不到如此设置的样式。

style 指向只包含在 HTML 代码里用 style 属性声明的样式。但这几乎没有使用价值，因为样式应该与标记分离开来。

现在，你或许认为用DOM 去处理 CSS 样式毫无意义，但这里还有另外一种情况可以让 DOM style 对象能够正确地反射出我们设置的样式。你用 DOM 设置的样式，就可以用 DOM 再把他们检索出来。

使用 DOM 设置样式

通过 JS 设置样式并不难，不过，你或许应该先问问自己，为什么要这么做？

何时该用 DOM 设置样式？

到底是使用 CSS 还是使用 DOM

最简单的答案就是选用最容易实现的办法。比如说，如果只是想让链接再鼠标指针悬停在其上是改变颜色，就应该使用 CSS。

伪类 hover 已经到的了绝大多数浏览器的支持，但是如果想利用这个伪类在鼠标悬停在其他元素上是改变样式，支持这种用法的浏览器就没有这么多了。

DOM 却能够得到公平的对待。绝大多数的现代浏览器，虽然对 CSS 伪类的支持很不完整，但对 DOM 却都有着良好的支持。

需要使用纯粹的 CSS 来解决，还是利用 DOM 来设置样式，你需要考虑一下因素：

1.这个问题最简单的解决方案是什么

2.那种解决方案会得到更多的浏览器的支持

要做出明智的抉择，就必须对 CSS 和 DOM 支持技术又足够深入的了解。如果你手里只有榔头，那么你看到的任何东西都像是的钉子。

想改变某个元素的呈现效果，使用 CSS；如果想改变某个元素的行为，使用 DOM 。如果你想根据某个元素的行为去改变它的呈现效果，请运用你的智慧。
